# AI非标数据清洗工坊 — UI/交互设计文档

> 版本: v0.1 | 状态: Draft | 最后更新: 2026-05-10

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **极简优先** | 目标用户非技术人员，每个页面只放一个核心操作 |
| **进度可见** | 所有异步操作（诊断、清洗、导出）都有实时进度反馈 |
| **操作可逆** | 执行清洗前可回退，每一步都可以回到上一步 |
| **容错设计** | 文件上传失败、格式不支持、编码错误 → 清晰的中文提示+解决方案建议 |
| **移动端兼容** | 上传和查看在手机上可用，复杂对比视图仅桌面端 |

## 4. 交互状态规范

### 4.1 加载状态

| 场景 | 表现 |
|------|------|
| 页面加载 | Skeleton骨架屏 (非Spin加载圈) |
| 上传中 | 文件图标 + 进度条百分比 |
| 诊断中 | 步骤条动画 + "正在分析第 X/15 列..." |
| 方案生成中 | 步骤条动画 + "AI正在分析数据..." |
| 执行清洗 | 步骤条动画 + 进度条 + "正在执行去重..." |

### 4.2 空状态

| 场景 | 表现 |
|------|------|
| 无历史任务 | 插画 + "还没有清洗过数据，开始你的第一个任务吧" + [开始]按钮 |
| 无模板 | 插画 + "还没有保存过模板，在清洗方案页可以保存" |
| 无诊断结果 | 正常等待诊断完成，不展示空状态 |

### 4.3 错误状态

| 场景 | 表现 | 用户操作 |
|------|------|---------|
| 文件格式不支持 | 红色警告横幅 + 格式要求 | 重新选择文件 |
| 文件过大 | 红色警告横幅 + 大小限制 | 重新选择或拆分文件 |
| 编码无法识别 | 弹窗 + 建议手动选择编码 | 选择GBK/UTF-8/Shift-JIS |
| 诊断失败 | 错误提示 + 重试按钮 | 重新诊断 |
| 清洗失败 | 错误提示 + 具体失败行号 | 查看日志、重试 |
| 网络错误 | 全局通知 | 自动重试3次后提示手动刷新 |
| 余额不足 | 弹窗 + 套餐升级入口 | 升级套餐 |

### 4.4 边界情况

| 场景 | 处理方式 |
|------|---------|
| 空文件 (0行) | 上传时校验行数，提示"文件为空，请检查" |
| 单列文件 | 正常诊断，减少分析维度 |
| 超大列数 (>50列) | 正常诊断，前端表格支持横向滚动 |
| 文件名为非ASCII | 保留原始文件名，存储时UUID重命名 |
| 同时上传多个文件 | MVP只支持单个，拖拽区域检测到多个时提示 |

## 6. 响应式断点

| 断点 | 宽度 | 适配策略 |
|------|------|---------|
| 桌面端 | >1024px | 全功能，对比视图左右分栏 |
| 平板 | 768-1024px | 全功能，对比视图上下分栏 |
| 移动端 | <768px | 上传 + 查看报告可用，对比仅显示统计摘要 |
